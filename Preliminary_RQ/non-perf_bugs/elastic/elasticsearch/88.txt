Here is a simple way to define the alias using the REST API (`localhost:9200/_aliases`):\n\n```\n{\n    actions : [\n        { add : { index : \"test1\", alias : \"alias1\" } }\n    ]\n}\n```\n\nIn the above, we define an `alias1` for index `test1`. In all operations, we can use `alias1` instead of `test1`. The API also allows to remove an alias from an index, and better yet remove and then add an alias for an index:\n\n```\n{\n    actions : [\n        { remove : { index : \"test1\", alias : \"alias1\" } },\n        { add : { index : \"test1\", alias : \"alias2\" } }\n    ]\n}\n```\n\nIn the above example, we simple remove `alias1` and add `alias2` to `test1` in a single, atomic operation.\n\nLast, but not least, an alias can be assigned to more than one index. This applies to APIs that accept more than one index, such as search and count. Instead of specifying a list of indices, we can just alias them to the same alias, and use it.

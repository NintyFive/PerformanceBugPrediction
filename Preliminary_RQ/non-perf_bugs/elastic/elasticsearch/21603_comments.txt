One which allows templating the whole search query, and a `template` query (deprecate) which allows you to embed a query inside the query DSL.\n\nThe first example you provide is using the `template` query, which results in the following request being executed  (note the double `query`):\n\n```\nGET /_search\n{\n  \"query\": {\n    \"query\": {\n      \"bool\": {\n        \"must\": [\n          {\n            \"match\": {\n              \"moId\": \"some_name\"\n            }\n          },\n          {\n            \"match\": {\n              \"metaClass\": \"some_class\"\n            }\n          }\n        ]\n      }\n    }\n  }\n}\n\n```\n\nThere used to be a `query` clause which allowed you to use a query inside a filter.
